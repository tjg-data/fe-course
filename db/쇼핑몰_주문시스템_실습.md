## 1. 쇼핑몰 주문 시스템 DB

  
### (1) 요구사항 정의
1. 회원 관리 (Member)<br>
회원 정보를 저장한다 (이름, 이메일, 가입일시)<br>
한 회원은 여러 주문을 할 수 있다<br><br>
2. 상품 관리 (Product)<br>
상품 정보를 저장한다 (상품명, 가격)<br>
주문되지 않은 상품도 목록에 포함되어야 한다<br><br>
3. 주문 관리 (Order)<br>
주문은 특정 회원에 의해 생성된다<br>
주문일자를 기록하며, 연·월·일 형식으로 조회 가능해야 한다<br><br>
4. 주문 상세 관리 (OrderItem)<br>
한 주문에는 여러 상품이 포함될 수 있다<br>
각 항목마다 수량과 주문 당시 단가를 기록한다 (가격 변동 대응)<br><br>
5. 조회 요구사항<br>
특정 회원의 이름·이메일·가입일·주문일을 조회할 수 있어야 한다<br>
상품별 주문 건수를 조회할 수 있어야 한다 (주문 없는 상품도 포함)<br>
회원별 주문 내역과 제품명·가격을 함께 조회할 수 있어야 한다<br><br>

### (3) 트리거 생성 실습
```
-- trigger 생성 : 여러개의 sql문 포함
/************************************************/
delimiter $$
create trigger trg_member_mid
before insert on [테이블명]
for each row
begin
declare max_code int;  --  'M0001'

-- 현재 저장된 값 중 가장 큰 값을 가져옴
SELECT IFNULL(MAX(CAST(right(mid, 4) AS UNSIGNED)), 0)
INTO max_code
FROM [테이블명]; 

-- 'M0001' 형식으로 아이디 생성, LPAD(값, 크기, 채워지는 문자형식) : M0001
SET NEW.mid = concat('M', LPAD((max_code+1), 4, '0'));

end $$
delimiter ;
/************************************************/
```
